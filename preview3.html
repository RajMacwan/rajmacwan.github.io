Dot Matrix · Build-Up Variants

How should the dots appear?

6 ways the dot portrait can build itself up, pause when complete, then dissolve and rebuild. Each dot is an individual SVG element — pure CSS animations. Loops indefinitely.

9A — Sequential (Reading Order)

Top-left to bottom-right · 6ms per dot

Dots appear one at a time in reading order. Like a printer scanning line by line. ~12 seconds to build completely. Methodical, precise.

9B — Ripple (Center Outward)

Radar sweep from center

Dots appear in an expanding ring from the center outward. Sonar / radar ping feel. Builds in ~2 seconds. Elegant and focused.

9C — Scan Line (Top → Bottom)

Horizontal scan line · 2.5s sweep

Dots appear as an invisible horizontal scan line sweeps down. Classic "printing" effect. Feels like a document being rendered in real-time.

9D — Random Chaos

Each dot independent · random delay

Dots appear in random order across the canvas. Feels like data deserializing / noise resolving into order. Slightly chaotic, organic.

9E — Columns (Left → Right)

Column-by-column wipe

Dots appear column by column left to right. Feels like a CRT screen refresh or old-school computer boot. Crisp vertical wipe.

9F — Spiral (Outside-In)

Spiral from edges to center

Dots appear in a spiral pattern from outside to inside. Organic, natural feel — like ink being drawn in a pattern. Most artistic.
